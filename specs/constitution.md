# Project Constitution

## Principles

1. The website must deploy as static HTML, CSS, and JavaScript on GitHub Pages.
2. Content must be easy to update through Markdown or small structured data files.
3. Published claims must be accurate, professional, and free of private internal details.
4. The design should be credible for a senior technical leader: fast, readable, responsive, and restrained.
5. Dependencies should stay minimal and conventional.

## Constraints

- No backend services.
- No database.
- No authentication.
- No sensitive data collection.
- No analytics or contact forms unless explicitly added later.
- No invented career claims.
